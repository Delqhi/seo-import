# Contributing to OpenSIN

Thank you for your interest in contributing to OpenSIN! We welcome contributions from everyone.

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/[repo-name].git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests: `npm test`
6. Commit: `git commit -m "feat: add your feature"`
7. Push: `git push origin feature/your-feature`
8. Open a Pull Request

## 📝 Commit Convention

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — New feature
- `fix:` — Bug fix
- `docs:` — Documentation changes
- `style:` — Code style changes (formatting, etc.)
- `refactor:` — Code refactoring
- `test:` — Adding or updating tests
- `chore:` — Maintenance tasks

## 🧪 Testing

- Write tests for new features
- Ensure all existing tests pass
- Run `npm test` before submitting PR

## 🔒 Security

- Do NOT report security issues publicly
- Email: security@opensin.ai
- See [SECURITY.md](SECURITY.md) for details

## 🤝 Code of Conduct

Be respectful, inclusive, and constructive. We're building something amazing together.
